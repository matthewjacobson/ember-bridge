✦ Ember × Ember Bridge

This page plays the role of the Ember editor: it finds the locally running Ember Bridge app, pairs with it, and sends an embroidery design to a machine on your WiFi. View source — every step is commented.

1 Detect the bridge

Polling GET /api/health — no token needed, so a site can tell "installed and running" apart from "needs install" before pairing.

Looking for Ember Bridge on 127.0.0.1:17831…

2 Pair with a token

In Ember Bridge, open Settings and copy the API token. Ember would store this once and reuse it forever.

3 Choose a machine

GET /api/machines returns machines the user saved in the bridge plus the latest scan results; POST /api/discover sweeps the local network (takes a few seconds).

4 Send a design

The design bytes go straight in the request body: POST /api/send?ip=…&filename=…. The bridge answers 202 with a job, then uploads in the background while we poll GET /api/jobs/{id}.
